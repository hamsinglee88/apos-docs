# 保税品如何代发？

> 文档地址：常见问题FQA/保税品如何代发.md


一．代发渠道创建与绑定

保税代发需要与不同的代发渠道进行对接，因此需要在Apos管理后台（Aposadmin.jieztech.com）创建代发渠道。代发渠道的渠道编码是固定的，每个渠道在对接时会指定一个编码。对接方式“外部主动对接”表示是有第三方系统来对接我们的开放接口，“Apos主动对接”表示是Apos系统主动对接第三方系统的接口。

代发渠道创建后，还需要对需要使用该代发渠道的商户进行绑定，绑定后该商户才能在一般贸易代发应用中看见对应渠道。（操作绑定的管理员账号需要是改商户的销售经理，否则无法选择该商户）

二．开通应用

商户想要使用一般贸易代发功能，需要在应用商店中开通对应的应用。应用开通后可以进行相应的配置，配置中生成的sessionkey就是第三方系统对接时该商户的key，因此sessionkey每个商户唯一。

![image.png](http://media-hk01.oss-cn-hongkong.aliyuncs.com/images%2Fjackli%40jieztech.com%2F%E4%BF%9D%E7%A8%8E%E5%93%81%E5%A6%82%E4%BD%95%E4%BB%A3%E5%8F%91.md%2Fa781e871-478d-4fa5-a62e-869a48013614_image.png?Expires=1824437666&OSSAccessKeyId=LTAIXyAB35iFpHM0&Signature=7Jzb31NzFHco%2FKjGZ86h5wySrGM%3D&x-oss-process=image%2Fformat%2Cpng)

三．商品绑定代发仓

开通一般贸易代发应用配置代发仓信息后，可以在商品列表对已经创建好的一般贸易商品进行绑定代发仓的操作。没有绑定代发仓就是普通的一般贸易商品，绑定代发仓后就是一般贸易代发商品。
![image.png](http://media-hk01.oss-cn-hongkong.aliyuncs.com/images%2Fjackli%40jieztech.com%2F%E4%BF%9D%E7%A8%8E%E5%93%81%E5%A6%82%E4%BD%95%E4%BB%A3%E5%8F%91.md%2F964d67e0-8ead-4390-ab3b-7c8b56310e57_image.png?Expires=1824437714&OSSAccessKeyId=LTAIXyAB35iFpHM0&Signature=lDG2IdawwfZqjxkLGMreVMdjpQQ%3D&x-oss-process=image%2Fformat%2Cpng)

四．支付单申报配置

由于保税订单需要进行三单申报，保税代发订单的订单和物流单申报由第三方系统完成。但是支付单数据是Apos收集，所以支付单数据由Apos负责申报。因此需要进行支付单申报配置。

支付单申报需要开通专门的‘支付单申报’应用，开通应用后可以对支付单申报的相关信息进行配置

![image.png](http://media-hk01.oss-cn-hongkong.aliyuncs.com/images%2Fjackli%40jieztech.com%2F%E4%BF%9D%E7%A8%8E%E5%93%81%E5%A6%82%E4%BD%95%E4%BB%A3%E5%8F%91.md%2F292882d0-7e56-4a72-8a28-569da1413869_image.png?Expires=1824437769&OSSAccessKeyId=LTAIXyAB35iFpHM0&Signature=6wolyp7PmrrLmOBGwggb%2Fa0zGIA%3D&x-oss-process=image%2Fformat%2Cpng)

五．对外开放接口权限开通

由于代发业务是与第三方平台对接，Apos系统内有一个对外开放平台，所有与外部交互的接口权限都是在该平台内管控，所以要使用一般贸易代发，需要在改平台内开通对应的接口权限。

首先需要在商户后台的‘系统管理-API开放平台’内，创建API账号，勾选对应的接口权限。

![image.png](http://media-hk01.oss-cn-hongkong.aliyuncs.com/images%2Fjackli%40jieztech.com%2F%E4%BF%9D%E7%A8%8E%E5%93%81%E5%A6%82%E4%BD%95%E4%BB%A3%E5%8F%91.md%2F587e7513-0762-4dd4-8120-ae0d861f3cbb_image.png?Expires=1824437802&OSSAccessKeyId=LTAIXyAB35iFpHM0&Signature=wRb0EW5iS6TKXePwD9EZwxgjANc%3D&x-oss-process=image%2Fformat%2Cpng)

六．库存同步

保税代发商品的库存不用商户维护，根据商品所属代发渠道的对接类型，由Apos主动定时同步或者第三方系统主动推送。每次同步/推送后，对应商品的库存会直接变成推送过来的数量，在Apos系统内会产生一条库存调整的流水记录。

七．订单同步

保税代发品的订单和商品库存一样，根据商品所属代发渠道的对接类型，有Apos主动推送或者第三方系统主动拉取。如果订单中只有一个代发渠道的商品，就是整单推送/拉取；如果订单中有多个代发渠道或者包含普通一般贸易商品，订单会根据代发渠道不同分成多个包裹，支付数据也会拆分，按包裹推送/拉取数据。

八．订单申报

保税代发订单交易完成后，Apos系统就会推送支付单数据到海关。支付单申报通过后，将订单包裹推送/拉取到第三方系统，由第三方系统进行订单申报。申报成功后，将包裹申报结果和发货物流信息推送给Apos系统，Apos系统接收申报结果和发货信息后，更改对应订单包裹的申报状态、物流信息和发货状态。

九．订单售后

一般贸易代发订单的售后流程目前暂时未做规划，因此需要商户与第三方系统线下进行沟通订单售后事宜。
